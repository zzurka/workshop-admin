## Cilj

Strukturisan izvor za labor linije na fakturi: ko je radio, koliko sati, po kojoj naplatnoj satnici. Danas labor linija na fakturi nema izvor — piše se rukom. Razdvajanje pojmova:

- `hr.time_entries` — **prisustvo** (clock-in/out, za HR/platu) — ostaje kako jeste
- `workshop.work_order_labor` — **naplativ rad po nalogu** (za fakturisanje) — novo
- naplatna satnica (šta plaća mušterija) ≠ trošak zaposlenog (šta piše u [1.3 compensations](1.3-payroll.md))

## Izmene šeme

### A. Nova tabela `workshop.work_order_labor`

Nova skripta: `20260330_1255_T_work_order_labor_DDL.sql` (posle work_orders 1250, pre work_order_parts 1260)

```
id             UUID          PK, uuidv7()
tenant_id      UUID          NOT NULL
work_order_id  UUID          NOT NULL   -- kompozitni FK → work_orders
employee_id    UUID          NOT NULL   -- kompozitni FK → employees (ko je radio)
work_date      DATE          NOT NULL DEFAULT CURRENT_DATE
hours          NUMERIC(6,2)  NOT NULL   -- CHECK (hours > 0); uneseno, ne mereno (npr. 2.5)
hourly_rate    NUMERIC(10,2) NOT NULL   -- naplatna satnica, snapshot pri unosu; CHECK (>= 0)
description    VARCHAR(500)             -- šta je rađeno (ide u opis fakturne linije)
notes          TEXT                     -- interno
audit kolone + is_deleted
```

- `UNIQUE (tenant_id, id)` — FK target za invoice_lines
- IX: tenant_id, work_order_id, employee_id, is_deleted

### B. `workshop.invoice_lines` — veza ka labor stavci (koordinacija sa planom 1.7)

Nova kolona `work_order_labor_id UUID` (nullable) + kompozitni FK `(tenant_id, work_order_labor_id)` → work_order_labor + IX. Labor linija na fakturi tada ima izvor istog ranga kao parts linija (`work_order_part_id`). Izmena ide u isti prolaz po `20260330_1320` skripti kao i izmene iz 1.7 i 2.1.

### C. `tenant.tenants` — default naplatna satnica

Nova kolona `default_labor_rate NUMERIC(10,2)` (nullable, CHECK >= 0) — aplikacija njome prefill-uje `hourly_rate` pri unosu rada. Izmena ide u isti prolaz po `20260317_1002` skripti kao fiskalna polja iz 1.7.

## Tok (aplikativna pravila)

Mehaničar (ili šef) unosi sate po nalogu tokom/posle rada. Pri fakturisanju: svaka nefakturisana labor stavka naloga → predlog fakturne linije (`description`, `quantity = hours`, `unit_price = hourly_rate`, PDV po pravilima 1.7). Kasnija promena satnice ne dira postojeće stavke (snapshot).

## Test plan

1. Rebuild prolazi.
2. Labor sa nalogom/zaposlenim drugog tenanta → kompozitni FK odbija; `hours <= 0` → CHECK odbija.
3. Fakturna linija vezana za labor stavku drugog tenanta → odbija.
4. Tok: 2 labor stavke → 2 labor linije na fakturi sa ispravnim iznosima.
