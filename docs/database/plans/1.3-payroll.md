## Cilj i obim

Vođenje zarada: istorija ugovorene zarade po zaposlenom (i satnica i mesečna plata), mesečni obračunski ciklusi i isplatni listići. **Obim v1 = evidencija (bruto/neto/dodaci/odbici), NE poreski obračun** — obračun doprinosa i poreza (PIO, zdravstvo, porez na zarade) po srpskim propisima radi knjigovodstvo/spoljni softver; ovde se evidentiraju rezultati.

### B. Nova tabela `hr.employee_compensations`

Nova skripta: `20260330_1015_T_employee_compensations_DDL.sql` (posle employees 1010)

```
id           UUID          PK, uuidv7()
tenant_id    UUID          NOT NULL
employee_id  UUID          NOT NULL   -- kompozitni FK → hr.employees
basis        VARCHAR(10)   NOT NULL   -- CHECK IN ('monthly','hourly') — mesečna plata ili satnica
amount       NUMERIC(12,2) NOT NULL   -- CHECK (amount > 0)
currency_id  SMALLINT      NOT NULL   -- snapshot tenant defaulta
valid_from   DATE          NOT NULL
valid_to     DATE                     -- NULL = važeća; CHECK (valid_to IS NULL OR valid_to >= valid_from)
notes        TEXT
audit kolone + is_deleted
```

- Partial unique: `uq_hr_employee_compensations_open ON (employee_id) WHERE valid_to IS NULL AND NOT is_deleted` — najviše jedna otvorena (važeća) zarada po zaposlenom. Preklapanje zatvorenih perioda sprečava aplikacija (EXCLUDE constraint bi zahtevao btree_gist ekstenziju — namerno izbegnuto, v. odluka 3).
- IX: tenant_id, employee_id, is_deleted
- `basis` kao VARCHAR + CHECK (presedan: `email_outbox.status`), ne codebook — dve vrednosti, ne administrira se.

### C. `hr.employees` (izmena u mestu: `20260330_1010`)

- IX: tenant_id, payroll_run_status_id, is_deleted

### E. Nova tabela `hr.payslips`

Nova skripta: `20260401_1030_T_payslips_DDL.sql`

```
id                 UUID          PK, uuidv7()
tenant_id          UUID          NOT NULL
payroll_run_id     UUID          NOT NULL   -- kompozitni FK → payroll_runs
employee_id        UUID          NOT NULL   -- kompozitni FK → employees
hours_worked       NUMERIC(7,2)             -- iz time_entries za period (satničari); NULL za mesečne
base_amount        NUMERIC(12,2) NOT NULL   -- osnovna zarada za period
bonus_amount       NUMERIC(12,2) NOT NULL DEFAULT 0   -- stimulacije/prekovremeni
deductions_amount  NUMERIC(12,2) NOT NULL DEFAULT 0   -- obustave
gross_amount       NUMERIC(12,2) NOT NULL   -- bruto
net_amount         NUMERIC(12,2) NOT NULL   -- neto za isplatu
currency_id        SMALLINT      NOT NULL
notes              TEXT
audit kolone + is_deleted
```

- `UNIQUE (payroll_run_id, employee_id)` — jedan listić po zaposlenom u obračunu
- CHECK: svi iznosi >= 0 osim base/gross/net > 0? — `base_amount >= 0`, `gross_amount >= 0`, `net_amount >= 0` (dozvoli 0 za neplaćeno odsustvo ceo mesec)
- IX: tenant_id, payroll_run_id, employee_id, is_deleted

## Tok (aplikativna pravila)

Kreiranje run-a za mesec → generisanje payslips iz važećih `employee_compensations` (+ `hours_worked` iz time_entries za satničare, umanjenja iz leave_requests po tipu odsustva) → korekcije → `approved` → `paid`. Posle `approved` listići se ne menjaju (aplikativno pravilo). Isplata plate se knjiži i kao trošak — veza ka `workshop.expenses` (kategorija `salaries` postoji u seed-u) ostaje aplikativna, bez FK.
